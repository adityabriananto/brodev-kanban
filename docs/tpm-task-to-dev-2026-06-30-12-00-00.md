# TPM Task: Authentication Overhaul (Email & Password)

**Date/Time:** 2026-06-30 12:00:00
**Status:** Approved for Development (Major Refactor)
**Requested by:** User (Homeowner)
**Assigned to:** Dev Team

## 1. Executive Summary
Karena sistem *Auth* berbasis PIN dirasa kurang kokoh untuk jangka panjang, Pemilik Bisnis memutuskan untuk merombak total sistem Otentikasi menuju standar industri: **Email & Password Login**, yang berlaku baik untuk Owner maupun ART.

## 2. Action Items untuk Database
- [ ] Buang (DROP) kolom `pin_code` dari tabel `users` (jika sudah sempat dibuat).
- [ ] Tambahkan kolom `email` (VARCHAR, UNIQUE) dan `password_hash` (VARCHAR).
- [ ] *Seed* ulang data `users`:
  - `bapak@rumah.com` (Role: owner, Password: Hash dari 'password123')
  - `ibu@rumah.com` (Role: owner, Password: Hash dari 'password123')
  - `art@rumah.com` (Role: art, Password: Hash dari 'password123')

## 3. Action Items untuk Backend
- [ ] *Install* library `bcrypt` untuk melakukan *hashing*.
- [ ] Ganti endpoint `POST /api/users/verify-pin` dengan `POST /api/auth/login` (Menerima `email` & `password`). Verifikasi dengan `bcrypt.compare`.
- [ ] Buat endpoint baru `PUT /api/auth/change-password` yang menerima `oldPassword` dan `newPassword` untuk merubah sandi pengguna.

## 4. Action Items untuk Frontend
- [ ] **Halaman Login:** Rombak halaman "Pilih Profil/Avatar" menjadi Form Login standar (Email & Password) bergaya *Glassmorphism* yang bersih. Hapus komponen `PinModal`.
- [ ] **Simpan Sesi:** Pastikan kredensial (atau profil login) tersimpan di `localStorage` agar tidak perlu *login* berulang kali.
- [ ] **Ganti Password UI:** Tambahkan tombol "Ubah Password" (ikon gembok) di pojok *Kanban Board* yang akan membuka *modal* pergantian *password*.

## 5. Catatan TPM
Ini adalah perombakan besar di sektor awal aplikasi (*Entry Point*). Pastikan *routing* Frontend (*React Router*) mengunci akses ke `/board` jika belum *login*. Tolong eksekusi perintah perombakan ini sekarang juga.
