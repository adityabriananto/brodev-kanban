## 1. Phase 1 Objective
Membangun fondasi utama aplikasi **Brodev Kanban Board** tanpa fitur *real-time* atau *cloud deployment* terlebih dahulu. Fokus pada UI "Pilih Profil", pembuatan *database* lokal via Docker, API dasar, dan *UI Board* bergaya premium.

## 2. Technical Stack (Phase 1)
- **Frontend:** React (Vite) + Vanilla CSS
- **Backend:** Node.js + Express (API RESTful konvensional)
- **Database:** PostgreSQL (berjalan di Docker lokal)

## 3. Database Schema (PostgreSQL)
Tolong buat skema inisialisasi lokal (misal di folder `db/init.sql`):
- **Table `users`:**
  - `id` (UUID, PK)
  - `name` (String: e.g., "Bapak", "Ibu", "ART")
  - `role` (Enum: 'owner', 'art')
- **Table `tasks`:**
  - `id` (UUID, PK)
  - `title` (String)
  - `description` (Text)
  - `status` (Enum: 'todo', 'in_progress', 'done')
  - `assigned_hours` (Decimal/Float)
  - `created_at` (Timestamp)
  - `updated_at` (Timestamp)

## 4. Backend Requirements (Phase 1)
- [ ] Buat file `package.json` dan struktur folder dasar (misal `server.js`).
- [ ] Setup koneksi database ke PostgreSQL lokal.
- [ ] Buat API CRUD dasar:
  - `GET /users` (untuk halaman Pilih Profil)
  - `GET /tasks`
  - `POST /tasks`
  - `PUT /tasks/:id`
  - `DELETE /tasks/:id`
- *(Catatan: Socket.io akan ditambahkan di Phase 2. Gunakan HTTP response standar terlebih dahulu).*

## 5. Frontend Requirements (Phase 1)
- [ ] Inisialisasi React menggunakan Vite (`npm create vite@latest`).
- [ ] Buat **Halaman Otentikasi (Pilih Profil)**:
  - Layar sederhana yang menampilkan profil pengguna yang didapat dari tabel `users` ("Bapak", "Ibu", "ART").
  - Tidak perlu *password*, cukup klik nama untuk masuk (state disimpan di *localStorage/Context*).
- [ ] Buat **Board Component**:
  - Tampilkan 3 kolom: "To Do", "In Progress", "Done".
  - Desain harus menggunakan *Vanilla CSS*, *Mobile-First*, elemen membulat (*rounded*), dan warna pastel/kontras tinggi agar tombol terlihat sangat jelas bagi ART.
- [ ] Integrasikan Frontend dengan REST API Node.js. Fitur merubah status (misal tombol "Pindah ke Done") cukup memanggil `PUT` lalu memperbarui state lokal React.
