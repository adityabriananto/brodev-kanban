# TPM Task: User Management & Task Assignment Feature

**Date/Time:** 2026-06-30 12:10:00
**Status:** Approved for Development
**Requested by:** User (Homeowner)
**Assigned to:** Dev Team

## 1. Executive Summary
Owner membutuhkan kemampuan mengelola akun pengguna (tambah ART baru, hapus, reset password) langsung dari UI tanpa menyentuh kode/database. Selain itu, setiap kartu tugas harus bisa ditugaskan ke ART spesifik, agar tahu siapa yang bertanggung jawab.

## 2. Asumsi TPM (karena User Auto-Approve)
- Tugas yang TIDAK ditugaskan ke siapapun: **tetap tampil untuk semua ART**
- Password ART baru: **Owner yang set saat mendaftarkan**
- Satu tugas: **satu ART penanggung jawab** (single-select, bisa di-extend ke multi nanti)

---

## 3. Action Items: Database

- [ ] Tambahkan kolom `assigned_to` (UUID, NULLABLE, FK ke `users.id`) ke tabel `tasks`:
  ```sql
  ALTER TABLE tasks ADD COLUMN assigned_to UUID REFERENCES users(id) ON DELETE SET NULL;
  ```
  *(Nullable = tugas tanpa penanggung jawab masih bisa dibuat)*

---

## 4. Action Items: Backend

- [ ] **`GET /api/users`** — Sudah ada. Pastikan mengembalikan semua user termasuk ART (tanpa `password_hash`).
- [ ] **`POST /api/users`** — Modifikasi existing route:
  - Terima: `name`, `role`, `email`, `password` (plain text).
  - Lakukan `bcrypt.hash(password, 10)` sebelum disimpan ke DB.
  - Hanya bisa dipanggil oleh `role: owner` (validasi dari `userId` di body request).
- [ ] **`DELETE /api/users/:id`** — Buat endpoint baru untuk menghapus pengguna.
- [ ] **`PUT /api/users/:id/reset-password`** — Endpoint baru untuk Owner me-reset password ART:
  - Terima: `newPassword` (plain text), hash lalu simpan.
- [ ] **`PUT /api/tasks/:id`** — Pastikan field `assigned_to` (UUID atau null) bisa di-update.
- [ ] **`GET /api/tasks`** — Modify query agar JOIN dengan `users` dan mengembalikan `assigned_to_name` juga.

## 5. Action Items: Frontend

### Panel Kelola Pengguna (`/users` atau modal dari Kanban Board)
- [ ] Tambahkan tombol **"Kelola Pengguna"** (ikon 👥) di sudut Kanban Board. Hanya tampil untuk `role: owner`.
- [ ] Buka modal/halaman berisi tabel: Nama | Email | Role | Aksi
- [ ] Form "Tambah Pengguna" dengan field: Nama, Email, Role (dropdown: owner/art), Password Awal.
- [ ] Tombol "Hapus" per baris pengguna (dengan konfirmasi).
- [ ] Tombol "Reset Password" per baris pengguna.

### Penugasan Tugas di Kanban Card

- [ ] Kartu tugas di Board menampilkan **chip/badge nama ART** yang ditugaskan (jika ada).
- [ ] Tambahkan filter **"Tampilkan Tugasku"** di Kanban Board (toggle button). Jika ART aktifkan, hanya tampilkan kartu yang `assigned_to === currentUser.id` ATAU yang `assigned_to === null`.
