# TPM Task: Implement Phase 3 (In-App Toast Notifications)

**Date/Time:** 2026-06-30 11:31:37
**Status:** Approved for Development (Execution Phase 3)
**Requested by:** User (Homeowner)
**Assigned to:** Dev Team

## 1. Executive Summary
Melanjutkan fondasi *Real-Time* dari Phase 2, **Phase 3** berfokus pada pengalaman pengguna terkait penambahan/perubahan "Jam" (*assigned_hours*).
Tujuannya adalah memastikan bahwa saat Pemilik Rumah (atau ART) merubah jam pada kartu tugas, lawan bicaranya langsung menerima notifikasi visual (*pop-up/toast*) tanpa mengganggu pekerjaan yang sedang mereka lakukan di aplikasi.

## 2. Backend Requirements (Socket.io Event Payload)
- [ ] Modifikasi *logic* di endpoint `PUT /tasks/:id` (atau endpoint khusus edit jam jika dipisah):
  - Deteksi apakah *field* `assigned_hours` mengalami perubahan (dibandingkan dengan data sebelum di-*update*).
  - Jika ada perubahan pada jam, *emit* *event* WebSocket khusus, misalnya: 
    `io.emit('hourChangedNotification', { taskId: 123, taskTitle: "Bersihkan Dapur", changedBy: "Bapak", newHours: 3 })`.

## 3. Frontend Requirements (React UI & Listener)
- [ ] **Install Library Toast:** Pasang library notifikasi ringan dan cantik (misal: `react-toastify` atau `react-hot-toast`), ATAU buat komponen *Toast/Snackbar* kustom menggunakan *Vanilla CSS* dengan gaya *glassmorphism*.
- [ ] **Socket Listener:**
  - Dengarkan *event* `hourChangedNotification` di komponen *Root/Board*.
  - Jika event diterima, cek apakah pengguna yang sedang login (profil aktif) **berbeda** dengan orang yang mengubah jam (misal: Jika `changedBy` = "Bapak" dan profil saat ini adalah "ART").
  - Jika profilnya berbeda, panggil fungsi pembuat *Toast*.
- [ ] **Desain Notifikasi:**
  - Tampilkan pesan yang ramah dan jelas.
  - Contoh teks: `"Bapak mengubah jam pada tugas [Bersihkan Dapur] menjadi 3 jam."`
  - Pesan harus otomatis menghilang setelah 4-5 detik.
  - Sediakan tombol 'x' kecil untuk menutup notifikasi secara manual.
  - Jika memungkinkan, putar bunyi "ting" (*notification sound*) pendek dan ramah saat *Toast* muncul (Opsional).

## 4. Catatan TPM
Pastikan komponen Notifikasi ini tidak *spamming* atau membekukan layar (non-blocking UI). Setelah ini selesai dirakit, aplikasi internal Kanban rumah tangga kita sudah 95% berfungsi sepenuhnya! 
Tim dev, kabari jika Phase 2 dan Phase 3 ini sudah siap di- *merge* dan di-*test*.
